# Optuna Hyperparameter Tuning - Bash-First Workflow

## Quick Start

### Recommended: Bash Script (Efficient + Logging)
```bash
cd agent_simulations/
bash run_tuning.sh
```

### Alternative: Python Wrapper (Legacy)
```bash
python run_all_algorithms.py
```

---

## Architecture

### Three-Layer Approach
1. **`run_tuning.sh`** ← **Main orchestrator** (RECOMMENDED)
   - Manages execution of all 4 algorithms sequentially
   - Comprehensive logging per algorithm
   - Timing information, progress tracking
   - Error handling and result validation
   - Creates dated log files for audit trail

2. **`hier_agent_10k_optuna_scenario5.py`** ← **Core tuning engine**
   - Runs Optuna multi-objective optimization (100 trials)
   - Pareto front extraction for each algorithm
   - Saves best hyperparameters + Pareto front data
   - Generates individual algorithm visualizations

3. **`generate_comparison_plots.py`** ← **Results analysis**
   - Lightweight visualization generator
   - Creates comparison plots from JSON results
   - No re-running simulations
   - Can be run anytime after tuning completes

## Multi-Objective Optimization Explanation

### Why Pareto Front?
Your requirement: "increase avg throughput AND maintain low variance"

**Problem with single weighted objective:**
- Arbitrary weights: 0.4×phase3_mean + 0.6×(1-phase3_std)
- User doesn't know the true trade-offs
- May miss excellent solutions at extreme points

**Solution with Pareto front:**
- Optuna optimizes TWO objectives simultaneously
- Returns ALL non-dominated solutions
- Users see complete trade-off curve
- Balance score selects "best" but Pareto data available

### How It Works
```
Study created with:
  directions=['maximize', 'minimize']
  
Objective functions return:
  (phase3_mean, phase3_std)
  
Optuna finds all trials where:
  - No other trial has higher mean AND lower std simultaneously
  
Balance score calculation:
  norm_mean = (mean - mean_min) / (mean_max - mean_min)
  norm_std = (std - std_min) / (std_max - std_min)
  balance_score = norm_mean * 0.4 + (1.0 - norm_std) * 0.6
  
Result: All Pareto trials saved, best trade-off selected
```

## Troubleshooting

### "algo_name" is not defined
- Already fixed in current `hier_agent_10k_optuna_scenario5.py`
- Script adds `algo_name = args.agent` before use

### No Pareto front found
- Optuna needs at least a few successful trials
- Check log file: `logs/optuna_tuning/AGENT_*.log`
- Ensure `hier_agent_10k_optuna_scenario5.py` runs successfully alone:
  ```bash
  python hier_agent_10k_optuna_scenario5.py -a UCB -d test.db -n 2 -s 42
  ```

### Results file not found
- Check if Python script ran successfully: see log file
- Verify `arrays/optuna_10k_scenario5/` directory created
- Check for errors in individual algorithm logs

### Need to Resume?
- Optuna database persists in `optuna_dbs/`
- Rerun bash script - it will resume from trial count in DB
- Each algorithm tracks progress independently

## Configuration Customization

Edit `run_tuning.sh` for:
- `AGENTS`: Which algorithms to tune
- `N_TRIALS`: Trials per algorithm (default 100)
- `SEED`: Random seed (default 42)
- `RESULTS_DIR`: Where to save results
- `LOGS_DIR`: Where to save logs

Example: Run only UCB and EGreedy with 50 trials:
```bash
AGENTS=("UCB" "EGreedy")
N_TRIALS=50
```
